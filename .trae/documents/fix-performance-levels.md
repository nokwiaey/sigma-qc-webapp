# 修正性能等级和质控规则计划

## 问题分析

通过分析原始网页代码 (`ChartDisplay.js` 和 `main.js`)，发现当前实现与原始版本存在以下差异：

### 1. 性能等级划分（原始版本）

原始网页使用 **6个性能等级**：

| 性能等级 | σ 范围      | 颜色            | 质控规则                             |
| ---- | :-------- | ------------- | -------------------------------- |
| 世界一流 | σ ≥ 6     | #00ff00 (绿色)  | 1₃ₛ (N=2,R=1)                    |
| 优秀   | 5 ≤ σ < 6 | #008080 (青色)  | 1₃ₛ2₂ₛR₄ₛ (N=2,R=1)              |
| 良好   | 4 ≤ σ < 5 | #87CEEB (天蓝色) | 1₃ₛ2₂ₛR₄ₛ4₁ₛ (N=4,R=1或N=2,R=2)   |
| 临界   | 3 ≤ σ < 4 | #800080 (紫色)  | 1₃ₛ2₂ₛR₄ₛ4₁ₛ8ₓ (N=4,R=2或N=2,R=4) |
| 欠佳   | 2 ≤ σ < 3 | #800000 (深红色) | 无                                |
| 不可接受 | σ < 2     | #FF0000 (红色)  | 无                                |

### 2. 图表 σ 等值线（原始版本）

原始图表绘制 **σ = 2, 3, 4, 5, 6** 共5条等值线：

* σ = 6 线（世界一流边界）

* σ = 5 线（优秀边界）

* σ = 4 线（良好边界）

* σ = 3 线（临界边界）

* σ = 2 线（可接受/不可接受边界）

颜色顺序（从外到内）：`["#FF8080", "#FFFF00", "#00FF40", "#4032EB", "#969EED"]`

### 3. 图表上的质控规则标注位置

根据原始代码 `display_chart_and_zhushi` 函数：

* **世界一流区域** (CV/TEa: 0-10%): 标注 `1₃ₛ (N=2,R=1)`

* **优秀区域** (CV/TEa: 10-11%): 标注 `1₃ₛ2₂ₛR₄ₛ (N=2,R=1)`

* **良好区域** (CV/TEa: 12-13%): 标注 `1₃ₛ2₂ₛR₄ₛ4₁ₛ (N=4,R=1或N=2,R=2)`

* **临界区域** (CV/TEa: 16-18%): 标注 `1₃ₛ2₂ₛR₄ₛ4₁ₛ8ₓ (N=4,R=2或N=2,R=4)`

### 4. 批长度图（原始版本）

`display_chart_p_len` 函数显示批长度信息：

* 世界一流: 批长度=1000个患者样品

* 优秀: 批长度=450个患者样品

* 良好: 批长度=200个患者样品

* 临界: 批长度=45个患者样品

## 当前实现问题

1. **缺少"欠佳"等级**：当前只有5个等级，缺少 2 ≤ σ < 3 的"欠佳"等级
2. **σ 等值线数量不对**：当前可能只绘制了4条线，应该是5条（σ=2,3,4,5,6）
3. **颜色映射不正确**：需要与原始版本完全一致
4. **质控规则显示格式**：需要使用下标格式（如 1₃ₛ）
